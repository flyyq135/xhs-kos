# 经纪人使用流程

## 目标

让经纪人轻量输入、分阶段确认，最终得到有数据、有实地感、有专业判断的小红书内容。流程不要让经纪人一开始填复杂表格，而是先提示可选信息，再给计划和选题，确认后再写正文。

## 默认交互顺序

### 第一步：给使用提示和建议

先告诉经纪人：

```text
这条内容会按“经纪人专业判断”来做。你可以先补充以下信息，也可以不补充，我会先基于公开信息做一版逻辑计划。
```

建议补充项：

- 客户类型：首套、改善、带娃、老人同住、投资关注、卖旧买新
- 预算范围：总价、首付、月供承受区间
- 通勤方向：上班商圈、轨道线、是否开车
- 房源信息：小区、组团、楼栋、户型、面积、楼层、朝向
- 数据口径：在售量、挂牌、成交、同户型成交、成交周期
- 实地观察：噪音、车库、坡坎、商业人流、夜间灯光、出入口
- 经纪人判断：最想强调的优点、最想提醒客户避开的坑

如果经纪人没有补充，继续往下走，不要卡住。

### 第二步：先出逻辑计划和选题

正式写正文前，先输出：

```text
我的内容逻辑：
1. 客户画像假设
2. 核心信息差
3. 数据锚点计划
4. 实地体验切入点
5. 风险边界

选题方向：
1. ...
2. ...

我推荐先写第 X 个，因为...

请确认：
- 是否按这个主选题写？
- 是否要补充具体楼栋/户型/客户画像？
```

这一阶段只做计划和选题，不写完整笔记。

### 第三步：等经纪人确认

经纪人确认方式可以很简单：

- 可以
- 继续
- 按这个来
- 选第 3 个
- 改成改善客户
- 加上某栋楼/某户型

确认后再生成完整内容。

### 第四步：生成具体选题和笔记

正式内容包括：

- 推荐标题
- 封面文案
- 正式小红书正文
- 轮播分页
- 话题标签
- 评论区互动问题
- 经纪人内部核验表
- 实地看房清单
- 差异化评分
- 合规自检

正式笔记正文不要展示来源链接和平台出处，但必须自然出现 2-4 个数据锚点。

### 第五步：提示是否生成图片

正文完成后提示：

```text
下一步可以把这篇做成 3:4 小红书轮播图。建议先生成封面样图，确认风格后再生成全套正文图。
```

如果有多篇选题，问经纪人要生成哪一篇的图片。不要默认把所有选题都生成图片。

## 经纪人输入模板

```text
用房产小红书-分析类型笔记技能，帮我做【小区/板块/楼盘】内容。

客户类型：
预算：
通勤方向：
重点房源：
我想强调：
我担心的风险：
已有数据：
现场观察：

先给我逻辑计划和选题，确认后再写正文。
```

## 输出风格

对经纪人说话要像协作，不要像填表系统。

推荐语气：

```text
如果你暂时没有这些信息也没关系，我会先用公开信息做一版选题逻辑，后面你再补楼栋或户型，我可以继续细化。
```

避免：

- 一次要求经纪人补齐所有数据
- 直接跳过确认生成长文
- 输出过多理论解释
- 把内部核验来源写进正式笔记
- 生成图片前不确认是哪篇内容

## 质量标准

一条合格内容要满足：

- 有一个清楚的信息差
- 有 2-4 个自然出现的数据锚点
- 有 2 个以上实地体验或带看动作
- 有适合和不适合人群
- 有风险边界
- 有经纪人口吻
- 有下一步可执行动作
