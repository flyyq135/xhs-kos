# 内容框架与模板

## 选题矩阵

| 支柱 | 用户问题 | 推荐内容 |
|---|---|---|
| 板块优先级 | 同预算先看哪里 | 板块梯度、东西区差异、通勤半径 |
| 楼盘研判 | 这个盘值不值得看 | 定价、产品、竞品、风险、适配人群 |
| 政策翻译 | 新政策跟我有什么关系 | 变化、误读、影响对象、行动建议 |
| 土拍轨道 | 城市变化会影响哪里 | 供应、兑现周期、受益范围、限制条件 |
| 决策科普 | 怕买错、怕继续跌 | 预算、持有周期、替代方案、检查表 |
| 人群场景 | 我的家庭情况怎么选 | 首套、改善、婚前、学龄、养老 |
| 资料工具 | 想快速做功课 | 地图、户型、对比表、看房清单 |

## 系列设计

### 重庆买房优先级

1. 先定义预算、人群和持有周期。
2. 再讲板块内部的梯度，不做全城笼统排名。
3. 每个板块使用同一评价维度，方便横向比较。
4. 最后给替代板块和不适合人群。

评价维度可选：

- 总价门槛
- 轨道与驾车通勤
- 商业和生活成熟度
- 新房与二手供应
- 产品改善度
- 城市界面和地形
- 产业与人口支撑
- 未来供应压力
- 转手难度

### 楼盘定价研判

1. 项目一句话定位
2. 当前价格或待定价格的时间点
3. 与 2 至 4 个竞品做同口径对比
4. 土地、建面、精装、车位等口径说明
5. 户型、梯户比、景观、噪音、通勤等产品判断
6. 价格合理区间的依据
7. 超过或低于哪个条件时，结论会改变
8. 适合与不适合的人

### 政策误读纠正

1. 市场流行说法
2. 一句话判断：哪些对、哪些不对
3. 政策原文的适用对象和执行时间
4. 实际改变的变量
5. 没有改变的变量
6. 对首套、改善、卖房者分别有什么影响
7. 行动前还要向谁核实

### 购房焦虑拆解

1. 说出用户担心的最坏结果
2. 区分市场风险、房子风险和家庭现金流风险
3. 用持有周期和替代成本做判断
4. 给出“可以买、再等等、不要买”的条件
5. 提供当周能执行的三项动作

## 标题公式

只把公式当结构，不机械套词。

- `年份 + 重庆买房优先级｜板块名`
- `预算 + 人群 + 重庆板块怎么选`
- `项目名定价研判：先看这 4 个变量`
- `政策名别急着解读成房价上涨`
- `重庆买房怕继续跌，先回答这 3 个问题`
- `板块 A 和板块 B，适合的是两类家庭`
- `户型图拿到手，先检查这 5 个位置`
- `轨道开通不等于所有小区都受益`
- `我不建议这类家庭优先看项目名`
- `一张表看懂板块名的新房梯度`

### 楼盘信息差选题公式

具体小区/楼盘标题必须带变量，不用“值得买吗”“适合谁”这类空泛问题单独成题。

- `大家都只看【A】，但真正决定体验的是【B】`
- `同一个【小区名】，不同楼栋差距在哪里`
- `地图看着近，实际通勤卡在【变量】`
- `【小区名】适合【人群A】，不适合【人群B】`
- `买【小区名】前，必须晚上/高峰/雨天看一次`
- `【板块/小区】别只看【显性卖点】，还要看【隐性风险】`
- `【预算】看【小区名】，先比较这3类房源`
- `【小区名】的核心不是【共性优势】，而是【具体变量】`

避免标题：

- “闭眼买”
- “稳赚”
- “最后机会”
- “全重庆最值”
- “内部消息”
- “马上暴涨”
- 单独使用“值得买吗”
- 单独使用“适合谁”
- 单独使用“优缺点分析”

## 数据型笔记结构

具体楼盘、小区或板块研判优先使用这个结构：

1. 结论：一句话说清楚适合谁、不适合谁。
2. 3 个数据判断：挂牌/成交口径、产品/楼栋口径、交通/供应口径；对外写判断，不堆来源，但正文至少自然出现 2 个数据锚点。
3. 2 个实地观察：至少包含一个动线体验和一个居住干扰点。
4. 1 个容易踩坑点：例如只看商圈、忽略楼栋、忽略车库、忽略噪音。
5. 1 个看房清单：告诉用户到现场先核实什么。

如果实时价格或成交拿不到，内部必须写成“待核实”，但对外不要把“待核实”铺满正文。正式笔记要给用户下一步核实口径，例如“下定前一定看同面积段近阶段真实成交，不要只看挂牌”。同时补充可核验数字清单，如“组团参考价差、近90天同户型成交、楼栋到站厅步行时间”。

## 具体小区研判模板

```text
核心判断：
【小区名】不是简单的好或不好，关键看【变量1】、【变量2】和【变量3】。

数据口径：
- 价格/成交：挂牌区间、近90天成交、成交周期、同户型价差；内部无法核实时标待核实，对外转化成判断和核实动作。
- 产品/楼栋：分期、楼龄、户型面积段、梯户比、楼栋位置、朝向、临街/临商业/临轨道。
- 交通/空间：到轨道站真实步行、坡坎、红绿灯、换乘、车库出入口、早晚高峰拥堵。

正文数据锚点：
- 至少写出 2 个数字或范围，例如楼龄跨度、参考价差、在售量级、步行时间、换乘距离、面积段、梯户比。
- 不写来源链接，不写平台名堆砌。
- 数字后必须跟判断，例如“说明不能用一个均价看整个小区”。

实地体验：
- 工作日晚高峰走一次小区到轨道站。
- 晚上看一次临街、临商业、车库出入口和中庭噪音。

适合：
【人群、预算、通勤、持有周期】

不适合：
【对安静/新房感/低密/预算/学位确定性要求高的人】

行动清单：
下定前核实【3-5项具体动作】。
```

## 成熟大盘示例：谢家湾/二十四城类

这类内容不要只写“万象城旁、配套成熟、轨道方便”。更好的信息差是：

- 显性卖点：商圈近、轨道近、生活成熟。
- 真实变量：分期楼龄、楼栋临街/临商业、车库出入口、晚高峰人车流、到轨道站的真实步行、不同户型面积段和总价门槛。
- 数据锚点：可写组团建成年代跨度、公开参考单价差异、在售房源量级、谢家湾站换乘距离、楼栋到站厅实走时间等。
- 实地动作：工作日晚高峰从楼栋门口走到谢家湾站；夜间看商业灯光、人流和车流；从车库开到主干道；比较同预算新房、次新和老一点的二手房。
- 写作角度：`华润二十四城别只看万象城，楼栋位置才是关键`。

对外不要在正文末尾堆来源链接。可以写成：

```text
我看这类成熟大盘，会先把同面积段、同楼栋位置和近期真实成交拆开，再带客户走一次晚高峰动线。
```

内部保留来源、采样日期和核验表，方便客户追问时专业回应。

## 轮播模板

### 8 页板块优先级

1. 封面：范围明确的核心结论
2. 先说适合谁、不适合谁
3. 板块地图与通勤
4. 预算和供应梯度
5. 产品与生活配套
6. 竞品或替代板块
7. 风险、变量和待兑现事项
8. 看房顺序、核实清单、信息日期

### 7 页楼盘研判

1. 封面：项目名 + 核心判断
2. 项目定位和价格口径
3. 同板块竞品对比
4. 户型与居住体验
5. 通勤和配套
6. 风险与不适合人群
7. 现场看房检查表

### 6 页政策解读

1. 封面：政策影响结论
2. 原文改变了什么
3. 谁真正受影响
4. 常见误读
5. 对不同购房者的建议
6. 来源、日期和咨询渠道

## 口播模板

```text
开头：
如果你在重庆用【预算】买【板块/项目】，先别只看【常见卖点】。

结论：
我的判断是【结论】，但只适合【人群和条件】。

证据一：
先看【维度一】，因为【事实和影响】。

证据二：
再看【维度二】，和【竞品】相比，【差异】。

证据三：
最后看【维度三】，这里最容易被忽略的是【风险】。

边界：
如果你的【通勤/预算/持有周期】不同，结论可能相反。

行动：
现场重点核实【三项清单】。你更关心【二选一问题】？
```

## 标准内容日历

一周 4 篇的稳妥组合：

| 类型 | 数量 | 作用 |
|---|---:|---|
| 深度判断 | 1 | 建立专业心智 |
| 热点解读 | 1 | 获取时效流量 |
| 资料工具 | 1 | 提升收藏 |
| 人群场景 | 1 | 扩大受众和互动 |

月度规划时避免连续多篇只写同一项目。板块、项目、政策和人群内容要交替。

## 输出自评

每项 0 至 2 分，总分低于 14 分时重写：

- 搜索意图明确
- 重庆本地化
- 核心判断清楚
- 证据可追溯
- 风险边界完整
- 普通人看得懂
- 行动建议具体
- 合规
- 不照搬他人
